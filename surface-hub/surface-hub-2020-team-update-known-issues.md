# Problemas conhecidos: Windows 10 Team 2020 Update 

Este artigo lista problemas conhecidos do Windows 10 Team 2020 Update, o sistema operacional atual do Surface Hub.

Para garantir que o Surface Hub receba as atualizações **** mais recentes, entre com uma conta de Administrador e selecione Todos os aplicativos Atualização de Configurações e Segurança do Windows Update e instale todas as  >  ****  >  ****  >  **** atualizações.




| Problema                                                                                                   | Descrição                                                                                                                                                                                                                                                                                                                                                                                                                             | Solução                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| ----------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| As atualizações opcionais não de segurança (também conhecidas como versões "C" e "D" ) não estão sendo instaladas.            | Os aprimoramentos do Surface Hub geralmente são fornecidos por meio de atualizações não de segurança na 3ª terça-feira de cada mês ("versão em C"). Esse problema resulta em ter que aguardar até a segunda terça-feira do próximo mês (atualizações de segurança de versão B) para receber atualizações cumulativas específicas do Hub. | Nenhuma solução no momento.                                                                                                                                                                                                                                                                                                                                     |
| A autenticação de proxy falha quando a Conta de Dispositivo é inserida como domínio\alias (nas configurações do Active Directory).            | Quando a conta de dispositivo é configurada em OOBE/Settings/MDM usando o formato domínio\alias e o dispositivo está usando a Conta de Dispositivo para autenticação de proxy, a conta de dispositivo não é passada corretamente para o Proxy. Como resultado dessa configuração, o proxy não detecta a Conta de Dispositivo como um usuário válido para autenticar e rejeita a solicitação. Quando isso ocorre, o dispositivo não pode se conectar a redes internas ou externas. | Nenhuma solução no momento. Os Surface Hubs configurados para usar a conta de dispositivo para autenticação de proxy não devem instalar a atualização de 2020.                                                                                                                                                                                                                                                                                                                                                                                                |
| A autenticação de proxy falha para determinados dispositivos hub.                                                                        | Se os clientes configurarem o Surface Hub usando a Conta de Dispositivo para autenticação de proxy, a autenticação de proxy pode não funcionar. Quando isso ocorre, o dispositivo não pode se conectar a redes internas ou externas.                                                                                                                                                                                                                                       | Nenhuma solução no momento. Os Surface Hubs configurados para usar a conta de dispositivo para autenticação de proxy não devem instalar a atualização de 2020.                                                                                                                                                                                                                                                                                                                                                                                                |
| A sincronização de calendário falha quando o UPN e o endereço SMTP da conta do dispositivo não são iguales.                                                                        | Hubs estão tentando usar o endereço SMTP para autenticar em vez de UPN. Nesse cenário, o Exchange para de sincronizar reuniões com o dispositivo. Como resultado, o dispositivo não recebe novas reuniões.                                                                                                                                                                                                                                       | Os clientes podem configurar suas contas de dispositivo para que o UPN e o SMTP sejam iguais. Caso contrário, os dispositivos não devem instalar a atualização de 2020.                                                                                                                                                                                                                                                                                                                                                                                                 |
| A sincronização de calendário com caixas de correio locais falhará se o cliente não der suporte à Autenticação Moderna Híbrida.   | Os hubs estão tentando forçar a Autenticação Moderna Híbrida para dispositivos locais quando os clientes não habilitaram isso em suas topologias. Nesse cenário, o Exchange para de sincronizar reuniões com o dispositivo. Como resultado, o dispositivo não recebe novas reuniões.                                                                                                                                        | Os clientes podem habilitar a Autenticação Moderna Híbrida em seu Exchange Server local, mas somente se sua topologia permitir isso. Caso contrário, os dispositivos não devem instalar a atualização de 2020.<br> <br>**Saiba mais:** Como configurar o Exchange Server local para [usar a Autenticação Moderna Híbrida.](https://docs.microsoft.com/microsoft-365/enterprise/configure-exchange-server-for-hybrid-modern-authentication)                                                                                                |
| A entrada é impactada quando as configurações de política de Acesso Condicional são aplicadas.                                    | Quando um dispositivo é configurado usando uma configuração de política de Conformidade que exige informações no nível do dispositivo (por exemplo, "Exigir Firewall"), um Erro de Acesso Condicional 53000 é acionado. Quando isso ocorre, os usuários não podem entrar no dispositivo.                                                                                                                                                                                                 | Os clientes podem excluir as configurações de política de conformidade que exigem informações no nível do dispositivo da execução em dispositivos Surface Hub. Se isso não for possível devido a restrições de conformidade ou segurança, os dispositivos hub não devem instalar a atualização de 2020.<br> <br>**Saiba mais:** [Use políticas de conformidade para definir regras para dispositivos que você gerencia com o Intune.](https:/docs.microsoft.com/mem/intune/protect/device-compliance-get-started) |
| Os dispositivos hub 2S não podem receber atualizações de driver usando o WSUS.                                             | O Surface Hub 2S dá suporte ao Windows Update e ao Windows Update para Empresas para distribuir drivers; não há suporte para distribuição por meio do WSUS (Windows Server Update Services).                                                                                                                                                                                                                                                                      | Se estiver usando o WSUS, migre para o Windows Update para Empresas.<br> <br>**Saiba mais:** o [que é o Windows Update para empresas?](https://docs.microsoft.com/windows/deployment/update/waas-manage-updates-wufb)                                                                                                                                                                                                                                                                                                                            |
| O single sign-on falha para dispositivos ingressados no Azure Active Directory ao usar um telefone para entrar no Surface Hub | Se um administrador de IT tiver [](surface-hub-2s-phone-authenticate.md) configurado o locatário para usar a autenticação sem senha para entrar no Surface Hub e o dispositivo for ingressado no AAD, os usuários não poderão entrar usando um dispositivo móvel como um telefone.                                                                                                       | Entre manualmente no Surface Hub.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| A Central de Ações tem um link configurações não clicadas. | Esse link não deve aparecer no Windows 10 Team e pode causar confusão.   | A funcionalidade é a mesma de antes da atualização de 2020; a seção Aplicativos do menu Iniciar deve ser usada para iniciar o aplicativo Configurações.    |
| Os dispositivos Hub 2S relatam todos os 0s para o número de série após a atualização para 24 de novembro de 2020 para o Windows 10 Team 2020 para 2S. | Depois de instalar a atualização do Surface Hub 2S de 24 de novembro para o Windows 10 Team 2020 ou instalar a imagem de recuperação do Windows 10 Team 2020 (BMR), alguns dispositivos estão relatando todos os zeros para o número de série. Isso impede que os dispositivos se inscrevam em provedores MDM.  | This issue occurred when a device did not have **Surface SMC Firmware update 1.177.139.0** (part of the September 1, Atualização de 2020 para Windows 10 Team Creators Update) antes de passar para a atualização de firmware do **SMC 3.91.139.0** (incluída na BMR do Windows 10 Team 2020 ou parte da atualização de 24 de novembro de 2020 para o Windows 10 Team 2020). Como a atualização de firmware do **Surface SMC 3.93.139.0** agora é oferecida para dispositivos Surface Hub 2S por meio do Windows Update, não é mais possível encontrar esse problema. Uma BMR atualizada que contém esse firmware também estará disponível em breve. Os dispositivos já afetados terão um novo número de série gerado quando o novo firmware for instalado. Observe que o dispositivo precisará ser reinicializado duas vezes depois que a atualização for instalada para concluir a instalação. **Saiba mais:** histórico [de atualizações do Surface Hub.](surface-hub-update-history.md) |
